# Duro Launch Checklist

## Pre-Launch (1-2 days before)

### Repository
- [ ] README is complete and compelling
- [ ] License file (MIT) is present
- [ ] Installation instructions work on fresh machine
- [ ] Quick start guide is tested end-to-end
- [ ] GitHub repo description is set
- [ ] Topics/tags added (ai, mcp, memory, claude-code, agents)
- [ ] Social preview image uploaded (1280x640)

### Documentation
- [ ] Docs site builds without errors
- [ ] All internal links work
- [ ] Getting started guide is beginner-friendly
- [ ] At least one "wow moment" is documented

### Demo
- [ ] Rate limit scenario is seeded and working
- [ ] Demo can be replicated by others
- [ ] Screenshots/GIFs created (optional but helpful)

### Landing Page
- [ ] Landing page is deployed and accessible
- [ ] All links work (docs, GitHub, CTAs)
- [ ] Mobile responsive
- [ ] Fast load time

---

## Launch Day

### Hacker News
- [ ] Post Show HN between 8-10am PT (peak visibility)
- [ ] Title is under 80 chars, compelling
- [ ] Post body is ready (copy from show-hn.md)
- [ ] Have seed comments ready to add context
- [ ] Monitor and respond to comments for 2-3 hours

### Twitter/X
- [ ] Thread scheduled or ready to post
- [ ] Post 30-60 min after HN (cross-promote)
- [ ] Include GitHub link in first tweet
- [ ] Reply to your own thread with demo GIF if available
- [ ] Engage with replies

### Cross-Posting
- [ ] Reddit r/ChatGPT or r/ClaudeAI (if appropriate)
- [ ] Dev.to article (optional, can be day 2)
- [ ] LinkedIn post (optional)

---

## Post-Launch (first 24-48 hours)

### Engagement
- [ ] Respond to every HN comment
- [ ] Respond to Twitter replies
- [ ] Thank people who share/star
- [ ] Note feedback and feature requests

### Monitoring
- [ ] Watch GitHub stars/forks
- [ ] Monitor Issues for bugs/questions
- [ ] Check analytics if available

### Iteration
- [ ] Fix any bugs reported immediately
- [ ] Update docs based on common questions
- [ ] Consider quick wins from feedback

## Timing Recommendation

**Best launch window:**
- Tuesday, Wednesday, or Thursday
- 8-10am Pacific Time
- Avoid holidays and major tech news days

**Sequence:**
1. Post to HN at 8-9am PT
2. Post Twitter thread 30-60min later
3. Cross-link in HN comments
4. Monitor and engage for 2-3 hours

---

## Emergency Contacts

If something goes wrong:
- GitHub Issues for bug reports
- Be responsive, apologize, fix fast
- Have rollback plan if major issues

---

## Success Metrics

### Good launch:
- 50+ HN upvotes
- Top 10 on Show HN
- 100+ GitHub stars day 1
- Meaningful feedback/feature requests

### Great launch:
- 100+ HN upvotes
- Front page
- 500+ GitHub stars week 1
- Contributors/PRs from community

Remember: The goal isn't virality. It's finding early users who care about the problem and will give real feedback.
